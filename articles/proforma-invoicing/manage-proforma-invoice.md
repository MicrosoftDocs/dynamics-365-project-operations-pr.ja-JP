# <a name="manage-a-proforma-project-based-invoice"></a>プロジェクトの見積もりベース請求の管理

_**適用対象:** リソース/非在庫ベースのシナリオ向け Project Operations_

Dynamics 365 Project Operations では、見積送り状は Dynamics 365 Sales の請求書の拡張機能として作成されます。 ただし、請求に関しては、Sales と Project Operations の間で請求プロセスに多くの違いがあります。 たとえば、Project Operations では **請求書リスト** ページから新しい請求書を作成することはできませんが、Sales では作成することができます。 これらの違いおよび拡張機能は、販売注文に対する通常の請求書とは異なるプロジェクトの請求プロセスをサポートするために実装されています。

> [!IMPORTANT]
> このような違いがあるため、Sales と Project Operations における請求書の互換性はありません。

| **名前** | **概要** タブ | 既定ではプロジェクトの契約名に設定されます。 このフィールドは編集できます。 | 
| **通貨** | **概要** タブ | 既定では、プロジェクト契約の通貨に設定されています。 取り専用フィールドで、編集できないようにロックされています。 |
| **価格表** | **概要** タブ | 既定では、プロジェクト契約の価格表に設定されています。 取り専用フィールドで、編集できないようにロックされています。 | 
| **営業案件** | **概要** タブ | リンクされている営業案件の参照先です。 取り専用フィールドで、編集できないようにロックされています。 |

| **顧客** | **概要** タブ | リンクされているプロジェクト契約書の参照先です。 取り専用フィールドで、編集できないようにロックされています。 |
| **説明** | **概要** タブ | 請求書についての説明が記載されるテキストフィールドです。 このフィールドは編集できます。 | 
| **請求書送付先** および関連するフィールド | **概要タブ** | 既定値は、プロジェクト契約の顧客から設定されます。 このフィールドは編集できます。  | 
| **ステータス** | **概要** タブ | 次のオプションを設定します: **アクティブ**、**クローズ済み**、**支払済み**、および **キャンセル**。これらは編集することができます。 Project Operations で対応していないない状態には、**クローズ済み** と **キャンセル済** です。 </br> 請求書が作成されると、状態は **アクティブ** に設定されます。 </br>この状態は、請求書が確認された後にのみ **支払い済** と設定する必要があります。  |

| **割引金額** | **概要** タブ | このフィールドを編集して、割引額を入力できます。 このフィールドは、Project Operations の機能では対応していません。 このフィールドには対応していません。 |

## <a name="project-based-invoice-lines"></a>プロジェクトベースの請求書の品目

Project Operations では、プロジェクト契約の品目ごとに常に 1 つの請求書明細があります。 実績がない場合でも請求書明細が作成されます。 Project Operations の請求書品目のヘッダーには以下の情報が用意されています。

| フィールド | 場所 | 内容 | 
| --- | --- | --- |
| **請求書 ID** | **全般** タブ | 請求書 ID への参照です。 取り専用フィールドで、編集できないようにロックされています。 請求書 ID のリンクを使用して、請求書ヘッダーに戻ることができます。 | 
| **名前** | **全般** タブ | 既定で設定されている請求書明細の名前で、契約品目の名称に由来します。 このフィールドは編集できます。 |
| **Project** | **全般** タブ | 関連するプロジェクト契約品目上のプロジェクトです。 取り専用フィールドで、編集できないようにロックされています。 プロジェクト リンクを使用して、プロジェクトに移動できます。 | 
| **請求方法** | **全般** タブ | 関連するプロジェクト契約品目の課金方法です。 取り専用フィールドで、編集できないようにロックされています。 |
| **契約品目の金額** | **全般** タブ | プロジェクト契約品目に関連する契約金額です。 取り専用フィールドで、編集できないようにロックされています。 | 
| **請求期限** | **全般** タブ | この請求書のすべての請求書明細詳細の金額の合計です。 取り専用フィールドで、編集できないようにロックされています。 | 
| **金額** | **全般** タブ | この請求書の請求書の明細に記載されている全ての請求書の金額の合計額です。 取り専用フィールドで、編集できないようにロックされています。 このフィールドは、請求書ヘッダーの最終金額を計算する目的で使用されます。 | 
| **税額** | **全般** タブ | この請求書明細の詳細のすべての請求書明細の税額の合計です。 取り専用フィールドで、編集できないようにロックされています。 このフィールドは、請求書ヘッダーの最終税額を計算する目的で使用されます。 | 
| **小計** | **全般** タブ | この請求書行のすべての請求書行の明細の合計金額 (**税金 + 金額**) の合計。 取り専用フィールドで、編集できないようにロックされています。 このフィールドは、請求書ヘッダーの最終金額を計算する目的で使用されます。 |

## <a name="invoice-line-details"></a>請求明細行の詳細

プロジェクト請求書の各請求書行には、請求書明細の詳細が含まれています。 これらの明細の詳細は、請求書明細によって参照される契約品目に関連する未請求の売上実績とマイルストーンに関連しています。 これらのトランザクションはすべて **請求する準備ができました** とマークされています。

**時間と材料の請求書** 行の場合、請求書行の詳細は、**請求明細行** ページで **請求可能**、**請求不可**、および **無料** にグループ化されます。 **課金対象の請求書明細** の詳細は、請求書明細の合計に加算されます。 **無料** と **非課金の実績** は、請求書明細の合計に加算されません。

**固定価格請求書** 行の場合、請求書行の詳細は、関連する契約行で **請求準備完了** とマークされたマイルストーンから作成されます。 マイルストーンから請求書明細の詳細が作成されると、マイルストーンの請求状態は **顧客の請求書が作成されました** に更新されます。

## <a name="refresh-invoice-transactions"></a>請求明トランザクションの更新

請求書の作成後に提出された実績がある場合は、これらの実績を請求書に含めることができます。

1. **請求のバックログ ビュー** で、データを **請求する準備ができました** とマークします。   
2. ドラフトの見積請求書を開き、**アクション** リボンで、**請求明細行トランザクションの更新** を選択します。

  請求書明細の詳細は、過去の日付で **請求準備完了** とマークされているが、請求書には含まれない実績に対して作成されます。
